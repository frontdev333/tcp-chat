# TCP Chat Server

Простой и производительный TCP чат-сервер, написанный на Go. Сервер поддерживает множественные подключения, историю сообщений и предоставляет HTTP API для мониторинга состояния.

## Возможности

- 🚀 Поддержка множества одновременных TCP подключений.
- 💾 История чата (последние N сообщений).
- 📊 HTTP сервер для мониторинга (Health Check и статистика).
- ⚙️ Гибкая конфигурация через аргументы командной строки.
- 📝 Структурированное логирование (slog).
- 🛑 Graceful shutdown (корректное завершение работы).

## Установка и запуск

### Предварительные требования

- Go 1.21 или выше.

### Запуск из исходного кода

1. Клонируйте репозиторий (если применимо) или перейдите в папку проекта.
2. Запустите сервер:

```bash
go run ./cmd/server
```

Или с флагами конфигурации:

```bash
go run ./cmd/server -port 9000 -max-connections 50 -log-level DEBUG
```

### Сборка бинарного файла

```bash
go build -o tcp-chat main.go
./tcp-chat
```

## Конфигурация

Сервер настраивается с помощью флагов командной строки:

| Флаг | Тип | Значение по умолчанию | Описание |
|------|-----|-----------------------|----------|
| `-port` | string | `8080` | Порт для TCP чат-сервера. |
| `-max-connections` | int | `100` | Максимальное количество одновременных подключений. |
| `-log-level` | string | `INFO` | Уровень логирования (`DEBUG`, `INFO`, `ERROR`). |
| `-history-size` | int | `64` | Размер истории сообщений (должен быть степенью двойки). |

## Использование

### Подключение к чату

Вы можете использовать `telnet` или `nc` (netcat) для подключения к серверу:

```bash
nc localhost 8080
```
или
```bash
telnet localhost 8080
```

После подключения вы увидите историю сообщений и сможете отправлять свои сообщения в чат.

### Мониторинг

Сервер запускает дополнительный HTTP сервис на порту `8081` для мониторинга.

#### Эндпоинты

- `GET /health` - Проверка состояния сервера. Возвращает статус, количество активных соединений и время работы.
- `GET /stats` - Подробная статистика сервера.

Пример запроса:

```bash
curl http://localhost:8081/health
```

Пример ответа:
```json
{
    "status": "healthy",
    "active_connections": 5,
    "uptime_seconds": 120
}
```
